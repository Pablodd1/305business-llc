## 🚀 What Is 305business.llc?

305business.llc is a **full-stack business marketplace** connecting buyers and sellers of businesses across Miami-Dade, Broward, and Palm Beach counties. It is not just a listing site — it is an **AI-powered, self-optimizing marketplace** with three core intelligence layers:

1. **🤖 Smart Front Desk** — AI chatbot trained on the full site, acting as a receptionist that guides buyers and sellers
2. **🧠 Agentic Website Engine** — Self-optimizing SEO that adapts the site based on real user behavior
3. **🔥 Trending Heatmap** — Real-time social proof showing which businesses are getting the most attention

Built with a **blueprint aesthetic** inspired by technical drawings, it features real-time data, AI-powered matching, and self-optimizing SEO.

6. **Fallback** → If the Gemini API is unavailable, the bot answers from its built-in knowledge base (no API needed)

## 🧠 Agentic Website — What It Does For 305business

The **Agentic Website Engine** is a self-optimizing system that runs in the background of 305business.llc, constantly improving the site's visibility, conversions, and user experience without manual intervention. Here is what it specifically does for this business marketplace:

### 1. 🎯 Self-Optimizing SEO (No Manual Work)
**The Problem:** Business marketplaces live or die by search traffic. Most sites publish once and hope Google ranks them. 305business doesn't hope — it adapts.

**What It Does:**
- **Meta Title Optimization:** Tests variants like *"Miami's Premier Business Marketplace"* vs. *"Buy or Sell a Miami Business"* vs. *"Your Business Transaction Partner"* and keeps the one that gets more clicks
- **Meta Description Optimization:** Same A/B testing for descriptions — finds the hook that makes people click from Google
- **CTA Button Optimization:** Tests *"List Your Business"* vs. *"Browse Listings"* vs. *"Get Valuation"* and promotes the winner
- **Dynamic Keywords:** Tracks which search terms bring visitors and adjusts page content priority accordingly. If "buy restaurant Miami" is trending, the site surfaces restaurant listings higher

**Result:** Higher click-through rate from Google, more organic traffic, more listings and inquiries.

### 2. 🤖 AI Search Optimization (Google AI Overviews, ChatGPT, Perplexity)
**The Problem:** In 2026, people don't just Google — they ask ChatGPT, Perplexity, Claude, and Gemini. If your site isn't optimized for AI answers, you're invisible to a growing segment of buyers.

**What It Does:**
- **Structured Data Injection:** Automatically adds Schema.org markup (Organization, LocalBusiness, Service, FAQPage, HowTo) so AI systems can extract and cite 305business as a source
- **AISO Tags:** Special `<meta>` tags (`ai:description`, `ai:services`, `ai:pricing`, `ai:location`) that tell AI systems exactly what the site offers
- **Citation Optimization:** Formats content so AI systems can easily quote 305business as a trusted source for "buying a business in Miami"
- **Freshness Tracking:** Monitors content age and flags pages that need updating (e.g., "This blog post is 45 days old — refresh it")

**Result:** 305business appears as a cited source in ChatGPT, Google AI Overviews, and Perplexity answers about Miami business sales.

### 3. 📊 Conversion Tracking & Analytics
**The Problem:** You can't improve what you don't measure. 305business tracks every micro-conversion.

**What It Tracks:**
| Conversion | Value | Trigger |
|------------|-------|---------|
| Business Listing Submission | $50 | Form submitted on list-business.html |
| Buyer Inquiry | $25 | "Request Info" clicked on a listing |
| Broker Match Request | $75 | Broker matching form submitted |
| Valuation Request | $100 | Valuation form submitted |
| Consultation Booking | $30 | Booking link clicked |
| Trending Heatmap Click | $15 | User clicks a trending listing |
| AI Matching Use | $20 | User completes the AI matching form |
| Chatbot Engagement | $10 | User sends 3+ messages in chatbot |

**Result:** You know exactly which page, which CTA, and which listing generates the most revenue potential.

### 4. 🧪 A/B Testing (Automated)
**The Problem:** Gut feelings about what works are usually wrong. Data is right.

**What It Tests:**
- **Headlines:** "Miami's Premier Business Marketplace" vs. "Buy or Sell a Miami Business" vs. "Your Business Transaction Partner"
- **CTAs:** "List Your Business" vs. "Browse Listings" vs. "Get Valuation" vs. "Find a Broker" vs. "Contact Us"
- **Hero Images:** Different blueprint backgrounds vs. Miami skyline photos vs. business imagery
- **Trending Section Position:** Above the fold vs. below listings vs. in sidebar

**Result:** The site automatically converges on the highest-converting version. No designer or developer needed.

### 5. 🌐 Multi-Language SEO
**The Problem:** Miami is a multilingual city. English-only SEO misses Spanish, Russian, and Haitian Creole speakers.

**What It Does:**
- **Hreflang Tags:** Tells Google that `/es/`, `/ru/`, `/ht/` versions exist for every page
- **Localized Keywords:** Tracks Spanish search terms ("vender negocio Miami") and Russian terms ("купить бизнес Майами") separately
- **Language-Specific Schema:** Injects Spanish `LocalBusiness` schema for Spanish-language pages
- **Content Freshness Per Language:** Flags when each language version needs updating independently

**Result:** Spanish and Russian speakers find 305business in their native language Google searches.

### 6. ⚡ Performance Monitoring
**The Problem:** A slow site kills conversions. 53% of mobile users abandon a page that takes >3 seconds.

**What It Monitors:**
- **Core Web Vitals:** Largest Contentful Paint (LCP), First Input Delay (FID), Cumulative Layout Shift (CLS)
- **Page Speed:** Full load time, time-to-interactive
- **Alert Threshold:** If any page takes >3 seconds, it triggers an alert
- **Heatmap Tracking:** Tracks where users click, scroll, and drop off (can be enabled)

**Result:** The site stays fast. You get alerted before users complain.

### 7. 🔄 Content Freshness Engine
**The Problem:** Stale content hurts SEO. Google prefers fresh content.

**What It Does:**
- Tracks the age of every page, blog post, and listing
- Flags content older than 14 days (configurable) for refresh
- Suggests new blog topics based on trending searches
- Auto-generates market update summaries (when enabled)

**Result:** 305business always looks active and current to Google.

### 8. 🔗 Schema.org & Structured Data
**The Problem:** Without structured data, Google sees your site as plain text. With it, you get rich snippets, knowledge panels, and AI citations.

**What It Injects:**
- **Organization:** Company name, logo, URL, description, contact info
- **LocalBusiness:** Address, phone, service area (Miami-Dade, Broward, Palm Beach)
- **Service:** Each service (valuation, forecasting, pitch deck) as a structured service offering
- **FAQPage:** All FAQ questions as structured data for rich snippets
- **HowTo:** The 4-step buying process as a structured HowTo guide
- **BreadcrumbList:** Navigation breadcrumbs for better search results
- **ItemList:** Trending listings as a structured list

**Result:** Rich snippets in Google search, knowledge panel eligibility, and AI citation readiness.

### 9. 📡 Sitemap & Indexing Optimization
**The Problem:** Google can't rank what it can't find.

**What It Does:**
- **Dynamic Sitemap:** `sitemap.xml` with all pages, listings, and priority scores
- **Robots.txt:** Bot-friendly, allows GPTBot/ChatGPT-User for AI training visibility
- **Canonical URLs:** Prevents duplicate content penalties
- **Priority Scoring:** Higher priority for trending listings, lower for static pages

**Result:** Every page is indexed quickly. Trending listings get crawled more frequently.

### 10. 🎛️ Custom Events & Goals
**The Problem:** Google Analytics out-of-the-box doesn't track marketplace-specific actions.

**What It Tracks:**
- Search queries (what people are looking for)
- Filter usage (which categories are popular)
- AI Matching completions (how many people use the feature)
- Chatbot engagements (how many conversations happen)
- Heatmap interactions (which listings are getting attention)
- Listing detail views (which businesses are most viewed)
- Form abandonment (where users drop off)

**Result:** You know exactly how visitors interact with the marketplace, not just that they visited.

## 🔗 How the Three Systems Work Together

305business.llc is not just a website with a chatbot and some SEO. It is a **self-reinforcing marketplace intelligence system** where three layers feed each other:

```
┌─────────────────────────────────────────────────────────────┐
│  🤖 SMART FRONT DESK (Chatbot)                              │
│  • Answers questions, guides buyers/sellers                │
│  • Routes users to listings, services, booking               │
│  • Captures intent ("I want to buy a restaurant")            │
│  • Sends data to → Agentic Engine                          │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  🧠 AGENTIC WEBSITE ENGINE                                  │
│  • Analyzes chatbot conversations (what people ask about)   │
│  • Adjusts SEO: if "restaurant" is trending, surface it      │
│  • A/B tests CTAs based on what chatbot users click         │
│  • Updates meta titles based on popular search queries      │
│  • Feeds insights to → Heatmap ranking                      │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  🔥 TRENDING HEATMAP                                        │
│  • Shows what buyers are actually viewing (social proof)    │
│  • Creates urgency: "If others are interested, I should be"│
│  • Tracks clicks, views, inquiries → feeds back to chatbot  │
│  • Chatbot uses heat data to say "This one is trending"     │
└─────────────────────────────────────────────────────────────┘
                              ↓
                              ↓ (loop back to chatbot)
```

### The Feedback Loop

1. **Buyer visits site** → Chatbot greets them, asks what they're looking for
2. **Buyer says "I want a restaurant under $500K"** → Chatbot routes them to #listings and explains the AI Matching feature
3. **Buyer browses, clicks 3 restaurants** → Heatmap records clicks, those restaurants get warmer
4. **Agentic Engine sees** that "restaurant" and "under $500K" are popular → adjusts homepage to show more restaurant listings and updates meta title to include "Restaurants for Sale Miami"
5. **Next buyer finds the site** via Google search for "restaurants for sale Miami" → sees trending restaurants with heat indicators → trusts the site more → clicks more → heatmap gets hotter → more social proof → more conversions

### The Seller Side

1. **Seller visits site** → Chatbot greets them, asks if they want to sell
2. **Seller asks "How much is my business worth?"** → Chatbot explains valuation ($495), links to valuation page
3. **Seller submits listing** → Heatmap starts tracking their listing (initially NORMAL)
4. **Seller upgrades to featured** → Heat score gets +20 bonus, listing appears as WARM/TRENDING
5. **Buyers see the trending listing** → more clicks → heatmap gets hotter → more buyers → higher chance of sale
6. **Agentic Engine sees** that "dental practice for sale" is trending → creates blog post about selling dental practices → more SEO traffic → more sellers → more listings → more buyers → network effect

## 🏗️ Architecture

### Frontend
- **Pure HTML/CSS/JS** — no framework, maximum performance
- **Tailwind CSS** via CDN (utility classes)
- **Font Awesome** icons
- **JetBrains Mono** + **Inter** fonts
- **Blueprint aesthetic** — technical drawing style with grid backgrounds, corner brackets, dimension lines
- **Responsive:** Mobile-first with breakpoints at 768px and 480px

### Backend API
- **Python/Flask** server (`backend/api_server.py`)
- **Port:** 8087 (configurable via `PORT` env var)
- **Supabase:** PostgreSQL database for listings, contacts, analytics
- **Endpoints:**
  - `GET /api/health` — Health check
  - `GET /api/listings` — All listings with filters
  - `GET /api/listings/<slug>` — Single listing + increments view count
  - `POST /api/listings` — Create new listing
  - `GET /api/trending` — Trending heatmap data (top by engagement)
  - `POST /api/track-click/<slug>` — Track a click and increment inquiry count
  - `POST /api/inquiries` — Submit buyer inquiry
  - `POST /api/contact` — Submit contact form
  - `POST /api/valuation` — Submit valuation request
  - `POST /api/broker-match` — Submit broker match request
  - `POST /api/subscribe` — Email newsletter subscribe
  - `GET /api/sitemap.xml` — Generate XML sitemap
- **CORS:** Enabled for all origins
- **Error Handling:** Graceful fallbacks with descriptive messages

## 🚀 How to Run Locally

### Frontend (Static)
```bash
# Just open index.html in a browser, or serve with any static server
python3 -m http.server 8000
# Or use Live Server in VS Code
```

### Backend API
```bash
cd backend
pip install -r requirements.txt  # flask, supabase, python-dotenv
# Set environment variables:
export SUPABASE_URL=https://your-project.supabase.co
export SUPABASE_KEY=your-service-role-key
export PORT=8087
python api_server.py
```

### Supabase Setup
1. Create project at [supabase.com](https://supabase.com)
2. Run `305business_schema.sql` in SQL Editor
3. Copy Project URL and Service Role Key to `.env`
